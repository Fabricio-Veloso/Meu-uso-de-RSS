# Spec: atualizar docs iniciais do projeto

## Objetivo

Consolidar documentacao minima para continuar o projeto em sessoes curtas, sem depender do historico do chat.

## Contexto atual

Projeto ainda esta em fase de entendimento. Nao ha implementacao planejada no momento.

## Entrada

- `Docs/estudo-inicial.md`
- `Docs/templates/inventario-jogos.md`
- `Docs/templates/inventario-discord.md`
- `Docs/templates/inventario-fontes.md`
- `Docs/backlogs/produto/descobertas-e-decisoes.md`

## Saida esperada da proxima rodada

- Inventarios parcialmente preenchidos pelo usuario.
- Decisao inicial entre ferramenta pronta e FreshRSS local.
- Workboards atualizados com proximo bloco executavel.
- Se fizer sentido, atualizar `AGENTS.md` real do projeto a partir de `AGENTS(base).md`.

## Restricoes

- Nao construir software ainda.
- Nao adicionar stack tecnica sem necessidade.
- Preferir solucao pronta/local existente antes de projeto proprio.
- Manter documentacao curta e operacional.

## Criterio de aceite

- Existe fonte unica de contexto do estudo.
- Existem templates claros para preencher.
- Workboards indicam proxima acao.
- A proxima sessao pode continuar lendo poucos arquivos.

## Plano rapido para atualizar docs do projeto

1. Ler `Docs/estudo-inicial.md`.
2. Preencher ao menos 5 jogos em `Docs/templates/inventario-jogos.md` ou copiar unidades para um arquivo ativo.
3. Preencher principais canais em `Docs/templates/inventario-discord.md`.
4. Preencher 5 a 10 fontes em `Docs/templates/inventario-fontes.md`.
5. Decidir primeira ferramenta de teste: FreshRSS local, Inoreader ou Feedly.
6. Atualizar `Docs/tracking/WORKBOARD.md` com o bloco escolhido.
7. Atualizar backlog de produto com decisoes firmes.
